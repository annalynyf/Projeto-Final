Git


Git é um sistema de controle de versão de arquivos. Com eles podemos desenvolver um projeto onde várias pessoas podem contribuir, editar e criar novos arquivos ao mesmo tempo, permitindo que eles existam sem o risco de reverter as alterações.


Exemplos de Git


    Git clone
    - É uma comando para baixar o código-fonte existente de um repositório remoto.

    Git status
    - Geralmente este comando vai permitir ver quais arquivos estão sendo “rastreados” pelo git e quais modificações já foram enviadas para o stage.

    Git branch
    - É usado para verificar todas as branches presentes no repositório.

    Git add
    - esse comando é utilizado para adicionar os arquivos solicitados ao ambiente de stage.

    Git commit
    - Faz gravação em si das modificações, e cria um snapshot do estado atual do nosso projeto.

Onde posso usar o Git?


A gente pode usar o Git para qualquer projeto com diferentes tipos de arquivos, que podem ser código, texto, imagens, vídeo, áudio, etc. Que tem como seus principais objetivos permitir o controle de histórico e versão desses projetos, melhorar o trabalho em equipe e o fluxo de trabalho, oferecer segurança aos arquivos e muitas outras vantagens mencionadas neste artigo.


Vantagens de usar o Git


• O git tem um desempenho muito bom, e pode ser considerado um dos melhores softwares de controle de versão.


• Ele possui flexibilidade, e uma das grandes vantagens do Git é que a gente consegue adaptar formatos de trabalho não lineares e ainda dá rastrear cada uma dessas ramificações.


• O git possui uma boa segurança e uma das prioridades é a integridade do código fonte do seu projeto. Todas as informações, conteúdos de código, versões, commits, são tudo protegido com SHA1, que é um algoritmo seguro.
