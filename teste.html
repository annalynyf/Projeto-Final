O que vem a ser a sigla TDD?


TDD é uma sigla para Test Driven Development, ou Desenvolvimento Orientado a Testes. A ideia do TDD é para que a gente consiga trabalhar em ciclos. Geralmente estes ciclos podem ocorrer na seguinte ordem:


Primeiro, a gente vai escrever um teste unitário que inicialmente ele vai falhar, tendo em vista que o código ainda não foi implementado; Aí a gente vai ter que fazer um código que satisfaça esse teste, ou seja: implemente a funcionalidade em questão. Essa primeira implementação deverá satisfazer imediatamente o teste que foi escrito no ciclo anterior; Quando o código estiver implementado e o teste satisfeito, refatore o código para melhorar pontos como legibilidade. Logo após, deveremos executar o teste novamente. Vale lembrar que a nova versão do código também deverá passar sem que seja necessário modificar o teste escrito inicialmente.


É importante lembrar também que TDD não é uma forma de escrever teste, mas muitas pessoas pensam que ele é, só que na verdade, ele nada mais é que uma metodologia para desenvolver e escrever o código.
